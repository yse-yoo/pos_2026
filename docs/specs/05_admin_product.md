# 商品管理画面 仕様書

**バージョン:** 1.0.0  
**作成日:** 2026-04-24  
**対象画面:** 商品管理画面 (`/products`)、商品登録画面 (`/products/create`)、商品編集画面 (`/products/:id/edit`)

---

## 1. 概要

本仕様書は、**店舗管理者が商品マスタを管理するための商品管理機能** を定義する。  
`backend/pos_2026/docs/screen_design.md` の商品管理画面・商品登録画面・商品編集画面、および `system_design.md` の SPA + API 構成を前提とする。

本機能の目的は、POSレジ画面で使用する商品を **一覧表示・登録・編集・削除（CRUD）** できるようにすることにある。  
商品はカテゴリに属し、レジ画面での表示や会計処理に必要な最低限の属性を持つ。

---

## 2. 画面の役割

| 画面 | URL | 主な役割 |
|------|-----|----------|
| 商品管理画面 | `/products` | 商品一覧の確認、登録・編集・削除の入口 |
| 商品登録画面 | `/products/create` | 新規商品を登録する |
| 商品編集画面 | `/products/:id/edit` | 既存商品の内容を更新する |

---

## 3. 利用者

| 利用者 | 権限 |
|--------|------|
| 店舗管理者 | 商品の閲覧・登録・編集・削除 |

一般利用者や会計担当者向けの画面ではなく、**商品マスタ管理を行う管理者向け画面** とする。

## 5. 商品管理画面 UI仕様

### 5.1 画面ヘッダー

| 要素 | 内容 |
|------|------|
| 画面タイトル | `商品管理` |
| 補足文 | 「レジで使用する商品を管理できます」など |
| 新規登録ボタン | 商品登録画面 (`/products/create`) へ遷移 |

### 5.2 絞り込みエリア

| 要素 | 内容 |
|------|------|
| キーワード検索 | 商品名で部分一致検索 |
| カテゴリ絞り込み | 全カテゴリ / フード / ドリンク / トッピング / 有料袋 |
| 表示状態絞り込み | すべて / 表示中 / 非表示 |

### 5.3 商品一覧

| 列 | 内容 |
|----|------|
| ID | 商品ID |
| 商品名 | レジ画面に表示する名称 |
| 価格 | 税込の販売価格 |
| カテゴリ | 所属カテゴリ名 |
| アイコン | レジ画面カード用の絵文字または簡易記号 |
| 表示状態 | レジに表示するかどうか |
| 並び順 | 商品表示順 |
| 操作 | 編集、削除 |

- 一覧は `並び順 ASC`、同順時は `ID ASC` を基本とする
- 金額は `¥1,234` 形式で表示する
- 削除前には確認ダイアログを表示する
- 空データ時は「商品が登録されていません」を表示する

## 6. 商品登録・編集フォーム仕様

### 6.1 入力項目

| 項目 | 型 | 必須 | 説明 |
|------|----|------|------|
| 商品名 | string | 必須 | レジ画面と履歴・明細に表示する名称 |
| 価格 | number | 必須 | 0以上の整数、税込価格 |
| カテゴリID | number | 必須 | 商品が属するカテゴリ |
| アイコン | string | 任意 | 商品カードに表示する絵文字または短い記号 |
| 表示状態 | boolean | 必須 | `true` の場合、レジ画面で選択可能 |
| 並び順 | number | 任意 | レジ画面の商品並び順 |

### 6.2 バリデーション

| 項目 | 条件 |
|------|------|
| 商品名 | 1〜50文字、前後空白は除去 |
| 価格 | 0以上999999以下の整数 |
| カテゴリID | 既存カテゴリのいずれかであること |
| アイコン | 0〜4文字程度の短い表示値 |
| 並び順 | 0以上の整数 |

### 6.3 ボタン動作

| ボタン | 動作 |
|--------|------|
| 保存 | 入力内容をAPIへ送信し、成功後に `/products` へ戻る |
| キャンセル | 入力を破棄して `/products` へ戻る |

## 7. CRUD仕様

### 7.1 一覧取得（Read）

| 項目 | 内容 |
|------|------|
| メソッド | `GET` |
| エンドポイント | `/api/products` |
| 用途 | 商品一覧の取得 |

### 7.2 商品登録（Create）

| 項目 | 内容 |
|------|------|
| メソッド | `POST` |
| エンドポイント | `/api/products` |
| 用途 | 新規商品の登録 |

### 7.3 商品更新（Update）

| 項目 | 内容 |
|------|------|
| メソッド | `PUT` |
| エンドポイント | `/api/products/:id` |
| 用途 | 既存商品の更新 |

### 7.4 商品削除（Delete）

| 項目 | 内容 |
|------|------|
| メソッド | `DELETE` |
| エンドポイント | `/api/products/:id` |
| 用途 | 既存商品の削除 |

### 7.5 商品詳細取得

編集画面初期表示のため、以下も必要とする。

| 項目 | 内容 |
|------|------|
| メソッド | `GET` |
| エンドポイント | `/api/products/:id` |
| 用途 | 編集対象商品の取得 |

## 8. カテゴリ連携仕様

商品登録・編集では、カテゴリ選択肢が必要なためカテゴリ一覧取得を行う。

| 項目 | 内容 |
|------|------|
| メソッド | `GET` |
| エンドポイント | `/api/categories` |
| 用途 | カテゴリ選択肢の取得 |

カテゴリ自体の新規作成・編集・削除は `screen_design.md` 上では **カテゴリ管理画面 (`/categories`)** の責務とし、本仕様書では商品管理に必要な参照項目のみ扱う。

---

## 9. 必要な商品項目

POSレジ画面、売上履歴、売上詳細での商品利用を踏まえると、商品マスタには最低限以下の項目が必要である。

| 項目 | 必要性 | 用途 |
|------|--------|------|
| `id` | 必須 | 商品の一意識別 |
| `name` | 必須 | レジ画面・履歴明細で表示 |
| `price` | 必須 | 会計金額計算 |
| `categoryId` | 必須 | カテゴリ別表示・絞り込み |
| `icon` | 任意 | レジ画面の商品カード表示 |
| `isActive` | 必須 | レジ画面へ表示するか制御 |
| `sortOrder` | 任意 | レジ画面の表示順制御 |
| `createdAt` | 必須 | 管理データ監査用 |
| `updatedAt` | 必須 | 更新日時確認用 |

### 補足

- 在庫数、原価、仕入先、軽減税率区分は現時点ではスコープ外とする
- 将来的な拡張を見据え、`isActive` と `sortOrder` は早い段階で持っておく

## 14. 状態管理仕様

| 状態 | 型 | 説明 |
|------|----|------|
| `products` | Array | 商品一覧データ |
| `categories` | Array | カテゴリ選択肢 |
| `isLoading` | boolean | 一覧読込中 |
| `isSubmitting` | boolean | 登録・更新送信中 |
| `errorMessage` | string \| null | 画面表示用エラー |
| `searchKeyword` | string | 商品名検索条件 |
| `selectedCategoryId` | number \| null | カテゴリ絞り込み条件 |
| `selectedStatus` | string | 表示状態絞り込み条件 |

## 15. エラー・確認仕様

| ケース | 表示内容 |
|--------|----------|
| 一覧取得失敗 | 「商品一覧の取得に失敗しました」 |
| 登録失敗 | 「商品の登録に失敗しました」 |
| 更新失敗 | 「商品の更新に失敗しました」 |
| 削除失敗 | 「商品の削除に失敗しました」 |
| 削除確認 | 「この商品を削除しますか？」 |

削除済み商品が過去売上明細に影響しないよう、履歴側では売上時点の明細データを保持する前提とする。

---

## 16. デザイン方針

- 既存フロントと同じエコなグリーン系トーンを使う
- 管理画面でも視認性を優先し、文字はやや大きめにする
- 一覧は表形式を基本にし、操作ボタンは行末にまとめる
- 登録・更新の主要操作は強調色、削除は危険操作として区別する
- スマートフォンでは一覧をカード化して表示する

---

## 17. スコープ外

- 在庫管理
- 原価管理
- 仕入先管理
- 軽減税率・複数税率
- 画像アップロード
- 商品バリエーション管理

本仕様では、**POSで販売する商品を安全に管理するための基本CRUD** に範囲を限定する。
